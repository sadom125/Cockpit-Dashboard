## 开发注意事项 / 踩坑记录

### 1. Obsidian 子进程找不到 `node` ⚠️
- **现象**：点击按钮报错 `node: command not found`
- **根因**：Obsidian 的 `child_process.exec` 使用系统默认 shell（`/bin/sh`），**不加载 `~/.zshrc` 或 nvm 配置**，PATH 中没有用户安装的 node
- **修复**：代码中使用 `/Users/hang/.local/bin/node` 完整路径
  ```js
  const nodeBin = require('os').homedir() + '/.local/bin/node';
  exec('cd ' + serverDir + ' && ' + nodeBin + ' server.js', ...);
  ```

### 2. main.js 必须通过 build.js 构建
- `main.js` 是编译产物，由 `build.js` 拼接 `src/` 下各模块生成
- **开发流程**：改 `src/` 下的文件 → 执行 `node build.js` → Obsidian 刷新生效
- `build.js` 内的模块拼接部分是**模板字符串**（`const output = \`...\``），HTML 模板代码和 JS 混在一起，修改时注意转义

### 3. `_framework.js` 是参考副本
- `src/_framework.js` 包含了完整的 `CockpitView` 类定义，但它**不在构建列表中**
- 真正拼入 `main.js` 的是 `build.js` 中 `const output = \`...\`` 里的那段 HTML 模板代码（约 400 行），里面也有一份相同逻辑
- **需要同时在两处修改**：`build.js`（模板段）和 `src/_framework.js`（参考副本）
- 之前坑过：只改了 `_framework.js` 没改 `build.js`，导致构建后按钮没生效

### 4. 工具栏按钮超出宽度会换行
- 7 个按钮时 `flex-wrap: wrap` 可能导致布局不整齐
- 目前通过 `justify-content: center` + `gap: 10px` 保持居中

### 5. 文件路径必须用 `require('os').homedir()` 拼接
- 避免硬编码 `/Users/hang`，确保在不同机器兼容

### 6. Obsidian 加载插件机制
- 路径：`{vault}/.obsidian/plugins/cockpit-dashboard/main.js`
- 插件通过 `module.exports` 导出 `CockpitPlugin` 类
- **首次**安装需在 Obsidian 设置 → 社区插件中刷新列表并启用
- **非首次**修改后：禁用再启用插件即可生效（纯改代码不需要重装）

### 7. `child_process/exec` 的 `open` 命令
- macOS 上用 `exec('open http://...')` 打开浏览器
- `open` 命令系统自带，PATH 不愁

### 8. 备用的 `dashboard.backup-2026-06-06` 目录
- 目录在 `.obsidian/plugins/` 下，`manifest.json` 的 `id` 也是 `"cockpit-dashboard"`
- Obsidian 如果同时加载两个同 id 的插件可能冲突，但该目录名称带日期后缀，Obsidian 似乎按目录名区分
